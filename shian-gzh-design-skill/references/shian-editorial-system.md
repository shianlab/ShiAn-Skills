# Shian 公众号内容排版系统

> 适用于前 5 个 Shian 默认主题。这份文档管“怎么读”，主题文件管“长什么样”。冲突时以本文档的手机阅读和内容节奏规则为准。

## 手机阅读基线

- 正文、步骤、列表项：`16px / line-height:2`。
- 引言、金句：`17px / 1.9`，但不连续出现。
- 二级章节标题：`23–25px / 1.35–1.45`。
- 图注、英文标签、辅助提示：`12–15px`。
- 段后距默认 `18px`，章节前距 `42–48px`；不用小字或压缩行高换取一屏更多内容。
- 正文字重默认 `400–500`，强调 `700–800`；不让全文粗重。

## 内容节奏

1. 文章内标题只做一次，不加顶部口号、关注条、封面大卡。
2. 开头用 2–4 个自然段建立问题和阅读动机；原文没有金句时不硬造引言块。
3. `##` 承担主结构，`###` 优先用左竖线小标题或纯加粗文字，不用胶囊。
4. 目录只用于章节 ≥ 5、长清单或明显需要跳读的教程。
5. 1500–2500 字文章通常使用 1–3 个辅助块；超过 3000 字可放宽到 2–4 个。
6. 结尾先收束观点，CTA 与签名只在有真实文案时生成，不留占位符。

## 强调规则

- 先尊重原文的 `**加粗**`、`==高亮==`和下划线，再补充模型判断的重点。
- 默认每 100–150 字约 1 个强调，单段最多 2 个，相邻两句不连续强调。
- 优先标记结论、方法、关键数据、反常识判断；不标记普通主语、连接词和泛化名词。
- 同一句只用一种强调手段。下划线是默认，浅底高亮只用于全文最重要的 1–3 个判断。

## 五色主题的版式差异

| 主题 | 版式节奏 | 章节识别 | 辅助块气质 |
|---|---|---|---|
| 青瓷绿稿纸 | 均衡、稳定，适合方法论 | 数字 + 细横线 | 青瓷绿左竖线，陶土棕重点 |
| 暖杏手记 | 段落稍短、留白稍多，更叙事 | 纯中文标题 + 短横线 | 像页边旁注，不加类型标签 |
| 铅笔灰稿纸 | 信息更紧、层次更理性 | 左对齐数字 + 全宽细线 | 中性浅灰底，旧金只标结论 |
| 苔绿砂纸 | 节奏沉稳，章节间距更大 | 小号数字 + 苔绿短线 | 可用无底色左竖线，强调判断感 |
| 珊瑚米白 | 亲和、短段落，适合大众解释 | 珊瑚数字 + 米白分隔 | 玫瑰灰重点，不做可爱贴纸 |

五个主题可共用同一类 HTML 组件，但必须按上表调整章节标题、留白和辅助块用法；不能只做色值替换。
